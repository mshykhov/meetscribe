# Meetscribe

Automated meeting video processing on macOS. Drop a video, get a transcript with speakers and an AI summary.

## How it works

Video in watch folder -> WhisperX transcribes -> pyannote identifies speakers -> Claude generates summary -> organized output folder

```
{date}-{topic}/
├── video.mp4
├── transcript.txt     # [00:15] SPEAKER_00: text...
└── summary.md         # topic, decisions, action items
```

Date comes from video metadata. Topic is generated by Claude.

## Quick start

```bash
brew install python@3.12 ffmpeg terminal-notifier
brew install --cask swiftbar  # optional

git clone https://github.com/mshykhov/meetscribe.git && cd meetscribe
python3.12 -m venv .venv && .venv/bin/pip install -e .

cp .env.example .env  # set HF_TOKEN, adjust paths
./scripts/install.sh install
```

HuggingFace setup: [create token](https://huggingface.co/settings/tokens), accept licenses for [speaker-diarization](https://huggingface.co/pyannote/speaker-diarization-community-1) and [segmentation](https://huggingface.co/pyannote/segmentation-3.0).

## Usage

**Automatic:** drop video into watch folder. **Manual:** `.venv/bin/python -m src.process /path/to/video.mp4`

```bash
./scripts/install.sh health      # check status
./scripts/install.sh retry       # retry failed
./scripts/install.sh reprocess /path/to/video.mp4
```

## Configuration

Key `.env` options: `HF_TOKEN` (required), `WATCH_DIR`, `OUTPUT_DIR`, `WHISPER_MODEL` (tiny/small/medium/large-v2), `LANGUAGE`, `CLAUDE_MODEL`.

Recommended model: `medium` (1-2min per 2min video, great quality). Models cached locally with offline mode.

## Stack

WhisperX (transcription) + pyannote.audio (speaker diarization) + Claude CLI (summary) + launchd (background service) + SwiftBar (menu bar) + terminal-notifier (notifications)
